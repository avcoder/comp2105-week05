CSS Transitions

A transition describes how a property should display change when given a different value.

(When to move) + (What to move) + (How to move)

(When to move) + (What to move) + (How to move)

Trigger + Animatable Prop. + Transition

transition-property

the property you want to change

see animatable properties

transition-duration

in seconds or milliseconds: 3s or 3000ms

Though milliseconds are actually a lot more robust to use in production code

If an animation is too fast to be interprested, it's cognitive load benefits can be lost

  • 100 ms, feels instantaneous
  • 100 ms ~ 1s, feels connected / 250 ~ 300ms is the animation sweet spot
  • 1s ~ 10s, disconnected (Perhaps needs a play button?)

transition-timing-function

Optional property that "cushions" the transition

Default is set to ease

transition-delay

Optional property that is the number of milli/seconds to delay the change before firing it

Default is set to 0

          

          transition-property: color;
          transition-duration: 2s;
          transition-timing-function: ease-in;
          transition-delay: 0.5s;

transition: color 2s ease-in 0.5s;

Does it have to be in this order?

transition-property: all is not very performant -- Don't do it!

It doesn't let the browser optimize for things that might change and that might result in the browser devoting too many resources

Easing

Easing, also known as "cushioning" in studio animation, describes an animation's rate of change over time.

For example, deceleration is actually ease-out. Acceleration is ease-in

CSS Transitions are good when you want:


  • Single fire

    if you only want something to happen once

  • Granularity

    If you want to animate one or two properties in a given state

  • Bulletproof

    If transitions aren't supported, the change happens anyway (it's just a jump cut)

  • Timings

    Control animation timing for transitioned properties.

Summary:


  • Defining a number of visual properties to be transitioned from one state to another is relatively simple, with good browser support.

  • Be sure to set up the transition property, timing function, duration, and delay on the base styles of the element to be affected when it is interacted with

  • Comparison of the various timing functions available in addition to making your own via cubic-bezier
